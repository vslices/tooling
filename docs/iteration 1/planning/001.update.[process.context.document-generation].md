# Actualización del Contexto del Proceso de Generación Documental

## Contexto de actualización

Anteriormente, se documentó el contexto actual de la generación documental en VSlices. Principalmente es un trabajo manual de creación, ajuste y mantenimiento de documentos Markdown.

Ahora, esta generación documental se mejorará volviendola un proceso apoyado por herramientas que generen esqueletos de la documentación base, sin reemplazar el criterio humano.

## Cambios propuestos

Actualizar el documento base para que el contexto ya no quede centrado únicamente en generación manual, ni en sus problemas.

El nuevo contexto debe preservar que la generación documental sigue naciendo desde una fricción real de documentación, y que deben mantenerse actualizados en base a las iteraciones de los experimentos de Alive Lab, pero que ahora tambien se apoya en una herramienta que facilita la construcción.

## Resultado esperado

Después de aplicar esta actualización, el documento `process.context.document-generation` debería explicar:

* que la generación documental ocurre dentro de VSlices Docs Standard
* que el proceso nace desde la necesidad de mantener documentos consistentes
* que los templates estructurados son una herramienta para apoyar el proceso
* que la generación produce una base editable, no contenido final

## Impacto

Esta actualización impacta principalmente a:

* `process.structure.document-generation`, porque la estructura del proceso debe reflejar generación desde template
* `flow.use-case.template-generation`, porque el caso de uso concreta la generación del documento

## Criterio de aplicación

Esta actualización puede aplicarse cuando Planning cierre el primer slice de generación documental.

Debe considerarse aceptada si el equipo mantiene como alcance actual la generación de documentos base desde templates estructurados.

Si durante Building se decide no usar templates estructurados, esta actualización debería retirarse o reformularse.


## Segmentos actualizados

| Segmento | Cambio | Descripción |
| --- | --- | --- |
| `Escenario` | update | El escenario deja de ser solo y pasa a ser documentado como parte de una linea técnica, dentro de la suite VSlices, especificamente en el producto Docs Standard. |
| `Situación actual` | update | Ahora la generación documental se apoya en templates estructurados definidos por el negocio. |

### Escenario

Estamos observando el proceso de generación de documentos, perteneciente al ecosistema de VSlices, en el escenario de Docs Standard.

### Situación actual

Los documentos se generan mediante uso de "templates", los cuales dan soporte a idiomas, niveles y definen el contenido perteneciente a cada documento.

Las estructuras nacen desde conversaciones, observaciones, iteraciones reales y necesidades detectadas mientras se trabaja en VSlices Docs Standard.

### Puntos de dolor

* repetición manual de front matter, secciones y preguntas guía
* riesgo de diferencias accidentales entre documentos del mismo tipo
* confusión entre tipo documental, scope y nivel de detalle
* dificultad para mantener versiones equivalentes entre idiomas
* riesgo de convertir tooling en producto antes de validar el primer caso
* dificultad para recordar qué segmentos aplican a L0 o L1
